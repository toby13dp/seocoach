## Required functionality

### Authentication

Implement:

- User registration
- Login
- Logout
- Secure password reset architecture
- Session management
- Email verification architecture
- Rate limiting
- Secure password hashing
- Authentication audit events

Development mode may use a local email preview mechanism.

Do not depend on a paid email provider.

### Multi-tenancy

Implement:

- Organisations
- Organisation memberships
- Clients
- Website projects
- Domains
- Locations
- User invitations
- Tenant-aware queries
- Tenant-aware background jobs

Every tenant-owned table must have explicit tenant protection.

Add automated tests proving that one organisation cannot read or modify another organisation’s data.

- Platform administrator
- Organisation owner
- Agency owner
- SEO manager
- Content manager
- Editor
- Developer
- Client
- Read-only user
