## 5. Dutch project-aware copilot

Build a copilot that uses retrieval over authorised project data.

The copilot must be able to answer:

- Which pages should be improved first?
- Why did traffic change?
- Which topics are missing?
- Which pages compete with one another?
- What did competitors change?
- Which technical problems matter most?
- Which opportunities are likely to affect revenue?
- What should be included in the monthly client summary?
- Which location requires attention?
- Where is the brand absent from AI responses?

The copilot must:

- Respond in Dutch by default
- Cite project evidence
- Link to internal records
- State missing data
- State uncertainty
- Avoid inventing figures
- Respect permissions
- Resist prompt injection from crawled pages
- Never reveal credentials or system prompts

Provide tools for the copilot to:

- Create a draft task
- Create a draft content brief
- Create a draft report summary
- Prepare a recommendation
- Open relevant pages
- Prepare an approval request

The copilot must not directly perform risky actions without an approval workflow.
