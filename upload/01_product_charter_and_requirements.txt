## 5. Safety and ethical data collection

Do not implement unauthorised scraping of search engines, AI platforms or protected services.

For search rankings and SERP data, use:

- Official APIs
- Licensed provider APIs
- User-provided exports
- CSV imports
- Manual records

For AI visibility, clearly distinguish between:

- GEO readiness
- Externally measured AI visibility
- User-provided AI responses
- API-based AI visibility tests
- Local simulations

- robots.txt
- Rate limits

- User-configured allowlists

## 53. Completion rules

A feature is not complete when:

- It only has a non-functional button
- It only uses mock data outside an explicit demo mode
- It has no tenant protection
- It has no error state
- It has no loading state
- It has no empty state
- It has no tests
- It has no audit log for sensitive changes
- It presents a score without explanation
- It automates a risky action without approval
- It is inaccessible to keyboard users
- It is undocumented

Create the product charter and requirements documents now.

Do not start broad feature implementation yet.

After creating the documents:

1. Summarise the main architecture implications.
2. Identify the largest implementation risks.
3. Identify requirements needing provider adapters.
4. Identify requirements that need legal or policy-aware handling.
5. Update REQUIREMENTS_TRACEABILITY.md with every major module.
6. Update IMPLEMENTATION_STATUS.md.
7. Commit the documentation if Git is available.
