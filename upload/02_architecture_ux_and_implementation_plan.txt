- Agency owner
- Client
- Developer

## 4. Design principles

Use:

- One primary action per screen
- Plain Dutch language
- Visible progress
- Safe defaults
- Explainable scores
- Contextual help
- Consistent priority labels
- Consistent status labels
- No unexplained acronyms
- Minimal initial configuration
- Advanced settings behind optional controls

Do not create dense enterprise screens as the default experience.
